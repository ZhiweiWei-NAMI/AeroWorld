# SUMO Accident API And Episode Alignment

This document fixes the SUMO-side goal for Donghu traffic generation. The
source of truth for ground traffic accidents is SUMO/TraCI first; downstream
episode events must align their capture boundary, trigger time, and visible
entities to the SUMO incident record rather than inventing a separate motion
story later.

## Goal

Complete all ground-traffic accident events through SUMO, run the 270s Donghu
traffic simulation, and export Python-readable files that are compatible with
truth-frame generation.

The design must stay aligned with existing episode event semantics:

- Accident intent is explicit, not inferred from display text.
- Event chains preserve causal order: condition -> conflict/incident -> response
  -> recovery or terminal hold.
- Capture boundaries are anchored at the true incident location and time.
- Vehicle motion is generated by SUMO, not by hand-written episode-only
  waypoint patches.
- The SUMO-to-UE coordinate route remains unchanged:
  `SUMO XY -> SUMO net lon/lat -> geojson local XY -> traffic_bundle affine fit
  -> UE truth XY`.

## Existing Episode Accident Taxonomy

The current event scripts contain these SUMO-relevant ground events:

| Episode family | Existing intent | SUMO accident class | Required observable |
| --- | --- | --- | --- |
| `L2-5_v1` | `traffic_or_facility_incident`, recovery | `traffic_light_all_red_fault` | all-red signal, queue buildup, police/manual recovery |
| `L3-1_v1` | `traffic_or_facility_incident`, `detour` | `lane_closure_roadwork` | blocked lane, detour path, barriers/cones in UE |
| `L3-3_v*` | `traffic_or_facility_incident`, `isolation` | `hazmat_isolation_zone` | emergency vehicle reaches perimeter, road access constrained |
| `L4-4_v*` | `collision_or_near_miss`, `vehicle_brake` | `uav_vehicle_impact_brake` | vehicle trajectory crosses incident point, hard brake/stop |
| `L4-6_v*` | `jaywalk`, `vehicle_brake`, `retreat` | `ped_vehicle_brake` | pedestrian crossing conflict, vehicle emergency braking |
| `L4-9_v*` | `collision_or_near_miss`, `vehicle_brake` | `vehicle_intersection_conflict` | two vehicles converge at intersection, simultaneous brake |
| `L4-10_v*` | `ambulance`, `yield` | `emergency_vehicle_priority` | ambulance priority passage, civilian yield/clearance |
| `L4-11_v*` | `traffic_or_facility_incident`, `safe_stop`, `hazard_hold` | `av_sensor_fault_stop` | autonomous vehicle slows, stops in lane, hazards on |
| `L4-7_v*`, `X3` | `traffic_or_facility_incident`, `ambulance` | `medical_response_dispatch` | ambulance route to medical incident, optional isolation tape |
| `L5-1_v*` | `weather_degradation`, `slowdown` | `weather_speed_degradation` | global or regional vehicle slowdown during rain |

UAV-only events such as geofence conflicts, C2 loss, GNSS spoofing, UAV-UAV
separation, facade proximity, and pad contention stay outside SUMO traffic
generation. They may still share the same capture boundary contract, but SUMO
does not control UAV flight or pad arbitration.

## TraCI Injection Strategy

Each SUMO accident class must be executable with explicit TraCI calls. The
runner should record the injected method in the output so the episode layer can
audit the causal source.

| SUMO accident class | Primary TraCI calls | Notes |
| --- | --- | --- |
| `traffic_light_all_red_fault` | `trafficlight.setRedYellowGreenState()`, `trafficlight.setPhase()` | Force all links red for a bounded interval; restore the original program/phase after recovery. |
| `lane_closure_roadwork` | `lane.setDisallowed()`, `lane.setMaxSpeed()`, `vehicle.rerouteTraveltime()` | Prefer lane restriction plus reroute. UE props remain visual assets spawned by episode event. |
| `hazmat_isolation_zone` | `edge.setDisallowed()` or `lane.setDisallowed()`, `vehicle.rerouteTraveltime()` | Use a bounded edge/lane closure around the isolation perimeter. |
| `uav_vehicle_impact_brake` | `vehicle.slowDown()`, `vehicle.setStop()`, `vehicle.setSignals()` | SUMO controls the vehicle brake/stop; the UAV debris/impact visual remains an episode asset. |
| `ped_vehicle_brake` | `vehicle.slowDown()`, `vehicle.setStop()`, `vehicle.setSignals()` | Pedestrian remains episode/UE-side for now; SUMO exports the vehicle reaction. |
| `vehicle_intersection_conflict` | `vehicle.addFull()`, `vehicle.setSpeedMode()`, `vehicle.setLaneChangeMode()`, `vehicle.slowDown()`, `vehicle.setStop()` | Spawn two controlled vehicles on converging routes and brake them near the conflict point. |
| `emergency_vehicle_priority` | `vehicle.addFull()`, `vehicle.setSignals()`, `vehicle.changeLane()`, `vehicle.slowDown()` | Ambulance and yielding vehicle are SUMO vehicles; UE visual state mirrors `signals`. |
| `av_sensor_fault_stop` | `vehicle.slowDown()`, `vehicle.setStop()`, `vehicle.setSignals()` | Record `fault_mode=sensor_fault`, then hold with hazard lights. |
| `medical_response_dispatch` | `vehicle.addFull()`, `vehicle.setSignals()`, `vehicle.rerouteTraveltime()` | Medical incident position may be non-road; route ambulance to nearest accessible lane. |
| `weather_speed_degradation` | `edge.setMaxSpeed()`, `vehicle.setMaxSpeed()`, optional `vehicle.slowDown()` | Weather itself stays episode-side; SUMO exposes traffic slowdown. |

Use `vehicle.moveToXY()` only for controlled alignment or recovery, not as the
normal motion source. The normal vehicle positions in exported truth data should
come from SUMO simulation state.

## Incident Anchor Contract

Each accident definition must include:

- `incident_id`
- `episode_scenario_id`
- `episode_event_id`
- `intent`
- `causal_chain_id`
- `start_s`
- `end_s`
- `anchor_truth_xy_m`
- `sumo_edge_id`
- `sumo_lane_id`
- `lane_position_m`
- `injection_method`
- `affected_vehicle_ids`
- `expected_observable_effect`

The episode generator should use this anchor to set:

- `capture_boundary.center_enu_m`
- `capture_boundary.source_entity_id`
- event trigger tick/time
- visible traffic actors for the event
- `event_trace` incident metadata

If the existing episode anchor is more than the allowed snap threshold from a
SUMO vehicle lane, relocate the episode capture boundary to the SUMO incident
anchor. Do not weaken the mapper or directly treat SUMO XY as UE XY.

## SUMO Output API

The SUMO runner should export one manifest plus one frame stream. The
simulation step is `0.1s` to match the existing 10Hz episode tick convention.
Exported traffic frames are sampled every `0.5s`, i.e. every `5` ticks, and
each sampled frame must contain both vehicle states and traffic light states.

### `sumo_traffic_manifest.json`

Required fields:

- `schema_name: "sumo_traffic_manifest"`
- `schema_version`
- `map_id: "donghu_road_topo"`
- `net_xml`
- `coordinate_mapper.fit_summary`
- `duration_s: 270`
- `step_length_s`
- `sample_period_s: 0.5`
- `sample_every_ticks: 5`
- `demand_profile`
- `incident_plan`
- `outputs`

The demand profile is fixed for the first production run:

- `0-90s`: ramp active vehicles from low density to 200.
- `90-180s`: hold near 200 active vehicles.
- `180-270s`: ramp active vehicles down.

### `sumo_traffic_frames.jsonl`

Each row is one simulation frame:

```json
{
  "schema_name": "sumo_traffic_frame",
  "schema_version": "v1",
  "map_id": "donghu_road_topo",
  "tick": 0,
  "sim_time_s": 0.0,
  "vehicles": [],
  "traffic_lights": [],
  "active_incidents": []
}
```

Each vehicle record must include:

- `vehicle_id`
- `vehicle_type`
- `route_id`
- `sumo_edge_id`
- `sumo_lane_id`
- `lane_position_m`
- `sumo_xy_m`
- `truth_position_enu_m`
- `truth_yaw_deg`
- `speed_mps`
- `accel_mps2`
- `signals`
- `dimensions_m`
- `source: "sumo_traci"`

Each traffic light record must include:

- `tls_id`
- `program_id`
- `phase_index`
- `state`
- `next_switch_s`
- `controlled_links`

Each active incident record must include the incident anchor contract above.

## Read-Only Feasibility Notes

The current map supports the required SUMO operations:

- `map.net.xml` has 1325 non-internal edges, 3383 lanes, and 3666 connections.
- 29 traffic light programs are present.
- SUMO 1.8 TraCI exposes the needed methods:
  `vehicle.addFull`, `vehicle.slowDown`, `vehicle.setStop`,
  `vehicle.setSignals`, `lane.setDisallowed`, `edge.setDisallowed`, and
  `trafficlight.setRedYellowGreenState`.
- Existing `SumoTruthCoordinateMapper` fit error is sub-millimeter against
  `traffic_bundle`, so the coordinate route is viable.

Some current episode anchors are already near SUMO vehicle lanes and can be
lifted directly: `L2-5`, `L3-1`, `L4-4`, `L4-6`, `L4-9`, `L4-10`, `L4-11`,
and `L4-7`.

Some anchors need relocation to SUMO-accessible incident loci before they can
be considered traffic-grounded: notably `L3-3` hazmat perimeter points and the
current `X3` medical-response region.

## Acceptance Criteria

The SUMO mainline is complete only when all of the following are true:

- The 270s Donghu simulation runs deterministically from a saved incident plan.
- The output manifest records the coordinate mapper fit summary.
- Every SUMO-controlled vehicle frame contains both SUMO coordinates and UE
  truth coordinates.
- Every ground-traffic accident event in the taxonomy has an explicit incident
  record and TraCI injection method.
- Episode capture boundaries and event trigger times are derived from SUMO
  incident anchors.
- Validators can distinguish SUMO-grounded vehicle motion from UAV/pad-only
  episode logic.
- No fallback path silently invents vehicle movement outside SUMO.
