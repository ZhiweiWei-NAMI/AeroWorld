# Project Structure

Read `HANDOFF_LOW_ALTITUDE_SEMANTIC_EVENT_CHAIN.md` first. This Unreal project is organized around UE editor-hook capture, truth-frame playback, and Donghu traffic scenario generation. AirSim is the RPC bridge at `127.0.0.1:41451`; it is not the formal image-capture backend.

## Source And Runtime

- `Config/LowAltitude/`
  Authoritative UE runtime configuration for map context, asset catalog, weather profiles, scenario objects, pedestrian navigation, and traffic bundle inputs.
- `Source/`
  The game module and editor/runtime validation UI.
- `Content/`
  UE maps and imported content used by the project.
- `ExternalAssets/`
  External source assets used by authored UE content.

## Plugins

- `Plugins/AeroSimHost/`
  UE-side Aero bridge, scene sync, asset placement, weather application, fixed-world capture, and the authoritative `simAero*` contract.
- `Plugins/PedestrianRuntime/`
  Managed pedestrian spawning, variant application, animation, movement, and crowd runtime behavior.
- `Plugins/SumoImporter/`
  Python scenario generation, scenario packages, episode playback, capture orchestration, map-source materials, and postprocess tooling.
- `Plugins/AeroWorldContent/`
  Reusable UE assets, Blueprints, data assets, props, vehicles, facilities, and scenario visuals.
- `Plugins/AirSim/`
  AirSim runtime and RPC compatibility support. Formal images are captured through the UE editor hook.

## Scenario Data

- `Plugins/SumoImporter/Scenarios/donghu_dense_uav_rain_fall/spec/`
  Authored scenario inputs.
- `Plugins/SumoImporter/Scenarios/donghu_dense_uav_rain_fall/scripts/`
  Scenario-specific generator entrypoints.
- `Plugins/SumoImporter/Scenarios/donghu_dense_uav_rain_fall/artifacts/`
  Current generated scenario package consumed by `episode_render_host.py`.
- `Plugins/SumoImporter/Scenarios/_seed_packages/`
  Minimal seed inputs required by current scenario generation.

## Generated Locally

The following directories are generated by Unreal, AirSim, capture runs, validation, or local tooling and should not be treated as source:

- `Saved/`
- `Intermediate/`
- `DerivedDataCache/`
- `Plugins/*/Intermediate/`

Large render outputs should be regenerated into an explicit `--output_dir` when needed. Formal capture output is `F:\aw_cap`, with `F:\aw_cap_summary.csv`.
